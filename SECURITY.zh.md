# 安全策略

> 🌐 语言: [English](SECURITY.md) | [中文](SECURITY.zh.md)

## 支持的版本

仅 taichi 的**最新发布版本**会获得安全修复。

| 版本    | 支持                |
| ------- | ------------------- |
| latest  | :white_check_mark:  |
| < 较旧  | :x:（请先升级）      |

## 报告漏洞

**请不要通过公开的 GitHub issue 报告安全漏洞。**

请通过邮件将疑似漏洞私下报告至 **security@tickraft.com**。

为便于我们分类与复现，请包含：

- 问题描述及其潜在影响。
- 复现该漏洞的逐步操作说明。
- 受影响组件与配置的范围。
- 建议的修复或缓解方案（如有）。

## 响应时间

- 我们将在 **72 小时**内确认收到你的报告。
- 我们将在 **7 天**内提供初步评估，包括对严重程度的初步判定与修复计划。
- 在问题解决或关闭之前，我们会持续向你同步进展。

## 披露政策

- 我们遵循协调披露流程。
- 一旦修复可用并发布，我们将通过
  [GitHub Security Advisory](https://github.com/tickraft/taichi/security/advisories)
  公开披露，并致谢报告者（除非报告者希望匿名）。
- 在修复发布之前，恳请你不要公开披露该漏洞。

## 不适用范围

第三方依赖中的漏洞不适用本策略。请直接向对应上游维护者报告。我们使用 Dependabot 跟踪并更新存在漏洞的依赖；你可以开一个普通 issue 请求紧急升级依赖。

## 致谢

我们感谢每一位负责任地披露安全问题的人。希望被致谢的报告者将在相关发布说明中获得致谢。
